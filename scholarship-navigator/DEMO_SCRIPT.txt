[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's trace how this agent works behind the scenes.
First, user queries enter our Security Checkpoint node which scrubs PII and checks for prompt injections.
If the check passes, control moves to the Scholarship Orchestrator agent.
The Orchestrator coordinates the workflow and calls our specialized sub-agents: the Search & Eligibility Agent and the Checklist & Deadline Agent.
These sub-agents connect to a local MCP Server to query our active scholarship database.
Once a candidate profile is matching, the flow pauses at the Human Verification node, verify_profile, using a RequestInput event to seek manual confirmation of the details.
Upon receiving a 'yes' confirmation, the final node, generate_final_report, executes to output the matched list, documents checklist, and deadlines. (pause)

[3:00 - SECURITY & MCP]
Our Security Checkpoint enforces strict boundaries by redacting PII like phone numbers and email addresses, and logs structured JSON logs at INFO, WARNING, and CRITICAL levels.
The MCP Server isolates data access, exposing tools like search_scholarships and get_application_deadlines, making the sub-agents modular and clean. (pause)
